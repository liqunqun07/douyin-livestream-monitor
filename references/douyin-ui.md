# 抖音页面结构参考（实测版）

## 首页登录检测

- **已登录标识**: 页面存在 `a[href*="/user/self"]` 元素
- **扫码登录**: 未登录时页面显示大尺寸二维码

## 搜索直播间

搜索 URL: `https://www.douyin.com/search/{品牌名}?type=live`

搜索结果中找包含"直播中"文本的卡片。

## 直播间页面布局

进入直播间后 URL 格式: `https://live.douyin.com/{直播间ID}`

小黄车（购物袋图标）通常在直播画面区域下方或右侧。

## 小黄车商品列表

- **"全部商品"按钮**: 在小黄车面板上，点击后展开完整商品列表
- **商品定位**: `[data-e2e="promotion-title"]` 属性标记每个商品标题
- **首次使用协议**: 可能出现"使用须知"弹窗，需点击"同意"按钮

## 商品详情浮层

点击商品后弹出详情浮层在右侧：
- **关闭按钮**: 右上角一个小 img 图标（缩小按钮）
- **位置**: x 坐标 > 790, y 坐标 < 50, 尺寸 < 40x40, cursor: pointer
- **典型坐标**: 约 (840, 2) 处 30x28 像素的 img
- **注意**: Escape 键被页面拦截，无法关闭详情

## 风控表现

- 连接重置: 刷新重试即可
- 验证码/滑块: 需要用户手动操作
- 页面跳转到 user/self: 说明点击位置不对，重新 navigate 到直播间 URL
- bytereplay 遮罩层: Playwright 原生 click 可能被遮罩层拦截，使用 JS dispatchEvent 绕过
